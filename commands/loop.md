description: Run the real OODA loop on a decision, action, or stuck problem.
---

Apply the `ooda` skill to the decision/action/problem in $ARGUMENTS (or the one in context). Run Boyd's real loop, not the 4-box cartoon:

1. **Observe** — measure, don't assume. Name the cheapest-falsifier probe / the telemetry to pull; note how prior actions changed the field.
2. **Orient** (spend the most here) — surface the stale models and bias steering the call (previous experience, cultural dogma). On a surprise or failing baseline, run one destruction-and-creation pass: break the old frame, synthesize a new hypothesis.
3. **Decide** — pick the single highest-leverage lever; state it as a hypothesis.
4. **Act** — ship it as a test and define what result re-enters Observe.
5. **Loop** — set the tempo: short, varied, revert-safe cycles that re-orient; cheapest-falsifier first.

Flag Orient-blindness if the move is about to be made on an un-reframed model. If the orientation is a proven repertoire, take the Orient→Act shortcut and say so.
